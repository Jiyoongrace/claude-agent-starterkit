# IRMS 권한 신청 절차 런북

## 개요

IRMS(IT Resource Management System)는 사내 시스템 화면 접근 권한을 관리하는 플랫폼입니다.
신규 화면 접근 또는 권한 변경 시 아래 절차를 따르세요.

## 권한 등급

| 등급 | 코드 | 설명 |
|---|---|---|
| 읽기 | READ | 화면 조회만 가능 |
| 읽기/쓰기 | READ_WRITE | 데이터 입력 및 수정 가능 |
| 관리자 | ADMIN | 전체 기능 + 사용자 관리 |

## 신청 절차

### 1단계: 신청서 작성

아래 항목을 준비합니다:
- 신청자 이름 및 사번
- 대상 화면 ID (예: `SCR_QUALITY_MGR`)
- 요청 권한 등급
- 신청 사유 (업무 필요성 명시)

### 2단계: 결재 라인

```
신청자 작성
  → 팀장 승인 (1영업일 이내)
  → IT 보안 담당자 확인 (2영업일 이내)
  → 시스템 관리자 등록 (1영업일 이내)
```

총 처리 기간: **최대 4영업일**

### 3단계: IRMS 시스템 등록

결재 완료 후 시스템 관리자가 `TB_SCREEN_AUTH` 테이블에 권한을 등록합니다.

## 주요 화면 ID 목록

| 화면 ID | 화면명 | 담당 부서 |
|---|---|---|
| SCR_QUALITY_MGR | 품질관리 화면 | 품질팀 |
| SCR_SLAB_DESIGN | 슬라브 설계 화면 | 생산기술팀 |
| SCR_ERROR_MONITOR | 에러 모니터링 대시보드 | IT운영팀 |
| SCR_ROLLING_PARAM | 압연 파라미터 설정 | 압연기술팀 |
